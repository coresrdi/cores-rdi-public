# Contributing to CORES RDI

Thank you for your interest in the CORES Red Dawn Initiative!

## How to Contribute

### Reporting Issues

- Use GitHub Issues to report errors, suggest improvements, or ask questions
- Provide as much detail as possible
- Tag your issue appropriately (bug, enhancement, question)

### Suggesting Content

- Open an issue with the `suggestion` label
- Describe the topic and which volume it relates to
- Include references or sources when possible

### Pull Requests

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-topic`)
3. Commit your changes with clear messages
4. Push to your fork and open a Pull Request
5. Reference any related issues

### Translation

We maintain content in both French and English. Translation contributions are welcome:

- Ensure technical terms are consistent with the glossary
- Maintain the same Markdown structure
- Note the source language in your PR description

## Guidelines

- Keep public content at **Level A (Public)** classification
- Do not include proprietary calculations or detailed design data
- Follow standard Markdown formatting
- Use clear, technical language

## Code of Conduct

All contributors are expected to maintain a respectful and professional environment. See our community standards for details.

## Questions?

Open a GitHub Discussion or Issue — we are happy to help.
